LESSONBuild it up, step by step

Easy — familiar from Day 1
STEP 1

Pick from a list — IN

On Day 1 you filtered with = and OR. When you want several values, IN is cleaner and reads better.

-- instead of: country = 'US' OR country = 'UK'
SELECT * FROM leads WHERE country IN ('US','UK');
Your turn
Get leads from the US, UK, or Canada.
Answer
SELECT * FROM leads WHERE country IN ('US','UK','Canada');
STEP 2

Filter a range — BETWEEN

BETWEEN a AND b is an inclusive range — both ends count. Works for numbers and dates.

SELECT * FROM campaigns WHERE budget BETWEEN 2000 AND 6500;
Your turn
Get leads created in July 2025.
Answer
SELECT * FROM leads
WHERE created_date BETWEEN '2025-07-01' AND '2025-07-31';
STEP 3

Match a pattern — LIKE

For "contains", "starts with", etc. % = any run of characters, _ = exactly one. So '%Q3%' means "has Q3 anywhere", 'Q3%' means "starts with Q3".

SELECT * FROM campaigns WHERE name LIKE '%Q3%';
Your turn
Get campaigns whose name starts with "Q3".
Answer
SELECT * FROM campaigns WHERE name LIKE 'Q3%';
STEP 4

...but case matters — ILIKE

In Postgres, LIKE is case-sensitive: '%email%' won't match 'Email'. Use ILIKE (case-insensitive) or wrap the column in LOWER().

SELECT * FROM campaigns WHERE channel ILIKE 'email';
Your turn
Find all Email-channel campaigns without caring about the capitalization.
Answer
SELECT * FROM campaigns WHERE LOWER(channel) = 'email';
-- or simply: WHERE channel ILIKE 'email'
Now the tricky part — missing data
STEP 5

The NULL problem

NULL means "unknown." The catch: any comparison to unknown returns unknown, not true — so WHERE company_size = NULL matches nothing, silently. You must use IS NULL / IS NOT NULL.

-- WRONG (returns 0 rows): WHERE company_size = NULL
SELECT * FROM leads WHERE company_size IS NULL;
Your turn
Find leads that are missing a company size (a real data-quality check).
Answer
SELECT * FROM leads WHERE company_size IS NULL;
STEP 6

Fill the gaps — COALESCE

COALESCE(a, b, ...) returns the first value that isn't NULL. Perfect for showing a friendly default instead of a blank.

SELECT lead_id, COALESCE(company_size, 'Unknown') AS size FROM leads;
Your turn
List every lead's size, showing "Unknown" where it's missing.
Answer
SELECT lead_id, COALESCE(company_size, 'Unknown') AS size
FROM leads;
STEP 7

The subtle trap

Here's what interviewers plant: status <> 'won' also excludes rows where status is NULL (because unknown ≠ 'won' is... unknown). To include the unknowns, you have to say so explicitly.

SELECT * FROM leads
WHERE status <> 'won' OR status IS NULL;
Your turn
Get every lead that is not "won", including any with an unknown status. (Our data has none NULL here — but write it the safe way anyway.)
Answer
SELECT * FROM leads WHERE status <> 'won' OR status IS NULL;
Shaping the output
STEP 8

Text functions

Clean up and transform strings: UPPER/LOWER (case), LENGTH (character count), TRIM (strip spaces), CONCAT(a,b) or a || b (join), REPLACE (swap).

SELECT UPPER(channel) AS chan, LENGTH(name) AS chars FROM campaigns;
Your turn
Show each campaign's name in ALL CAPS along with how many characters it has.
Answer
SELECT UPPER(name) AS name_up, LENGTH(name) AS chars
FROM campaigns;
STEP 9

Date functions

EXTRACT(MONTH FROM d) pulls a part out of a date; DATE_TRUNC('month', d) rounds down to the month start (you'll use this to group later); and you can subtract dates directly: CURRENT_DATE - created_date.

SELECT lead_id, EXTRACT(MONTH FROM created_date) AS mth FROM leads;
Your turn
For each lead, show the created_date and the month number it was created.
Answer
SELECT lead_id, created_date,
       EXTRACT(MONTH FROM created_date) AS mth
FROM leads;
Put it together
CHALLENGE

Combine everything

One query, using pieces from several steps. Take your time and build it clause by clause.

Your turn
Get leads from the US or UK, created in July 2025, that have a known company size — showing the size in caps.
Answer
SELECT lead_id, country, created_date, UPPER(company_size) AS size
FROM leads
WHERE country IN ('US','UK')
  AND created_date BETWEEN '2025-07-01' AND '2025-07-31'
  AND company_size IS NOT NULL;

WATCHGotchas to remember

💡 = NULL matches nothing — always IS NULL / IS NOT NULL. And <> quietly drops NULLs too.
💡 LIKE is case-sensitive in Postgres — use ILIKE or LOWER().
💡 BETWEEN is inclusive; for timestamps prefer >= start AND < next_day so you don't miss the last day's events.
