LEARNConcept primer

IN · BETWEEN · LIKE

IN ('US','UK') matches any value in a list (cleaner than = 'US' OR = 'UK'). BETWEEN a AND b is an inclusive range (works for numbers and dates). LIKE pattern-matches text: % = any run of characters, _ = exactly one. So '%Q3%' = "contains Q3", 'Q3%' = "starts with Q3".

NULL — the big one

NULL means "unknown," and any comparison to it returns unknown, not true. So WHERE company_size = NULL matches nothing — you must use IS NULL / IS NOT NULL. COALESCE(a, b, 'Unknown') returns the first non-null value; NULLIF(x, 0) returns NULL when x = 0 (used to avoid divide-by-zero).

String functions

UPPER(x) / LOWER(x) change case, LENGTH(x) counts characters, TRIM(x) strips spaces, CONCAT(a, b) or a || b joins text, REPLACE(x,'a','b') swaps, SUBSTRING(x FROM 1 FOR 3) slices. Case-insensitive matching: LOWER(col) = 'email' or Postgres's ILIKE.

Date functions

EXTRACT(MONTH FROM d) pulls a part (YEAR/MONTH/DAY/DOW). DATE_TRUNC('month', d) rounds a date down to the start of the month (great for grouping later). Date math works directly: CURRENT_DATE - created_date gives days elapsed.

4. Campaigns whose name starts with "Q3" core
Hint: put the % only at the end.
Check your answer

5. All "Email" campaigns, case-insensitive core
Hint: ILIKE (Postgres) or LOWER(channel) = 'email'.
Check your answer
SELECT * FROM campaigns WHERE channel ILIKE 'email';
-- or: WHERE LOWER(channel) = 'email'
6. Leads missing a company size core
Hint: IS NULL — never = NULL.
Check your answer
SELECT * FROM leads WHERE company_size IS NULL;
7. Every lead's size, showing "Unknown" where it's missing core
Hint: COALESCE the column with a fallback string.
Check your answer
SELECT lead_id, COALESCE(company_size, 'Unknown') AS size
FROM leads;
8. Bottom-of-funnel leads: status demo or won core
Hint: IN on status.
Check your answer
SELECT * FROM leads WHERE status IN ('demo','won');
9. Each lead's creation month number stretch
Hint: EXTRACT(MONTH FROM created_date).
Check your answer
SELECT lead_id, created_date,
       EXTRACT(MONTH FROM created_date) AS mth
FROM leads;
10. Campaign names in caps, with their length stretch
Hint: UPPER() and LENGTH().
Check your answer
SELECT UPPER(name) AS name_up, LENGTH(name) AS chars
FROM campaigns;

WATCHGotchas to note

💡 = NULL matches nothing. Use IS NULL / IS NOT NULL. And status <> 'won' excludes NULL rows too — for "not won, including unknowns" write status <> 'won' OR status IS NULL.
💡 LIKE is case-sensitive in Postgres. '%q3%' won't match 'Q3'. Use ILIKE or wrap in LOWER().
💡 BETWEEN is inclusive on both ends — and for timestamps, BETWEEN '2025-07-01' AND '2025-07-31' stops at midnight of the 31st, missing that day's events. Prefer >= '2025-07-01' AND < '2025-08-01' for timestamps.
