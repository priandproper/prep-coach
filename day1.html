LearnConcept primer

Read these alongside SQLBolt 1–6. Every query is SELECT columns FROM table plus optional clauses, always in this order: SELECT → FROM → WHERE → ORDER BY → LIMIT.

SELECT — choose columns

SELECT * = all columns; SELECT name, budget = just those. Start with * while exploring, name columns when you know what you want.

WHERE — filter rows

Keeps only rows matching a condition: WHERE channel = 'Email'. Combine with AND / OR. Text goes in single quotes; numbers don't. Operators: = <> (not equal) > < >= <=.

ORDER BY — sort

ORDER BY budget DESC (high→low) or ASC (low→high, the default). Sort by multiple: ORDER BY channel, budget DESC.

LIMIT — cap the rows

LIMIT 5 returns the first 5. Combined with ORDER BY it gives you "top N": ORDER BY budget DESC LIMIT 5.

DISTINCT — unique values

SELECT DISTINCT channel FROM campaigns collapses duplicates — the everyday "what are the possible values?" query.

Stretch operators (for queries 6–10)

BETWEEN · IN · LIKE · IS NULL

BETWEEN '2025-07-01' AND '2025-07-31' (inclusive range) · IN ('US','UK') (matches any in the list) · LIKE '%Q3%' (% = any characters, so this = "contains Q3") · IS NULL / IS NOT NULL (test for missing values — never use = NULL, it silently fails).

Practice10 queries on your data

Write each in DB Fiddle's "Query SQL" panel and Run. Try before peeking. Expand "Check your answer" only to verify.

1. All campaigns, most recent first core
Hint: no WHERE needed — just sort by start_date descending.
Check your answer
SELECT * FROM campaigns ORDER BY start_date DESC;
2. Only Paid Search campaigns core
Hint: filter channel with a single-quoted string.
Check your answer
SELECT * FROM campaigns WHERE channel = 'Paid Search';
3. The distinct list of channels you run core
Hint: DISTINCT on one column.
Check your answer
SELECT DISTINCT channel FROM campaigns;
4. Campaigns with budget over 5000, biggest first core
Hint: a WHERE and an ORDER BY ... DESC together.
Check your answer
SELECT * FROM campaigns WHERE budget > 5000 ORDER BY budget DESC;
5. Your 5 highest-budget campaigns (top-N) core
Hint: sort descending, then LIMIT.
Check your answer
SELECT * FROM campaigns ORDER BY budget DESC LIMIT 5;
6. Leads created in July 2025 stretch
Hint: created_date BETWEEN … AND … (dates in single quotes).
Check your answer
SELECT * FROM leads
WHERE created_date BETWEEN '2025-07-01' AND '2025-07-31';
7. Leads from the US or UK stretch
Hint: IN ('US','UK') is cleaner than two ORs.
Check your answer
SELECT * FROM leads WHERE country IN ('US','UK');
8. Bottom-of-funnel leads: status demo or won stretch
Hint: same IN pattern on status. (This is a real "hot leads" query.)
Check your answer
SELECT * FROM leads WHERE status IN ('demo','won');
9. Data-quality check: leads missing company_size stretch
Hint: IS NULL — not = NULL. Analysts get asked this constantly.
Check your answer
SELECT * FROM leads WHERE company_size IS NULL;
10. Campaigns whose name contains "Q3" stretch
Hint: LIKE '%Q3%' — the % wildcards match anything on either side.
Check your answer
SELECT * FROM campaigns WHERE name LIKE '%Q3%';

WatchGotchas to note

💡 Text needs single quotes, numbers don't. WHERE channel = 'Email' ✓ but WHERE budget = 8000 (no quotes).
💡 = NULL never works. Missing values need IS NULL / IS NOT NULL. This trips up almost everyone early.
💡 LIKE is case-sensitive in Postgres. '%q3%' won't match 'Q3'. (Use ILIKE for case-insensitive — a good one to note for Day 9 dialects.)

Jot every quirk you hit in your notebook — these become your Day-9 dialect notes and your personal cheat-sheet.
